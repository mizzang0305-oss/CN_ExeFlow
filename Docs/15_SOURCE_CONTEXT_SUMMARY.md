# 원문 기반 핵심 컨텍스트 요약

본 설계는 기존 운영계획과 사용자 확장 아이디어를 종합하여 구성됨.

## 운영계획서 핵심 취지
- 대표이사 지시사항을 전사 총괄 관리
- 부서별 분류 관리
- 신규 / 진행중 / 지연 / 종결 상태 표준화
- 주간 결산 중심 보고
- 숫자 데이터화로 전체 현황과 부서별 현황 동시 확인

## 사용자 확장 방향
- 지시사항 등록을 부서 단위로 운영
- 부서장이 지시 전달 및 이행 관리
- 행동 단위, 문서, 사진 증빙 등록
- 대표는 취합 대시보드에서 총 진행, 지침별/부서별 이행사항 확인
- 필요 시 세부 근거 자료 drill-down 확인

## 설계 결론
- 마스터 지시사항 + 부서 실행 + 행동 로그 + 증빙 + 결산 구조로 설계 잠금
