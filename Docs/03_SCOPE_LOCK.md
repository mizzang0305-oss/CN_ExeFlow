# 범위 잠금 문서

## 1. 이번 버전에서 반드시 완료할 것
- 대표 지시사항 마스터 관리
- 부서 배정 및 부서별 상태 관리
- 담당자 단위 행동 로그
- 증빙 첨부(사진/문서)
- 지시 상태 표준화
- 완료 요청 / 승인 / 반려 플로우
- 주간 결산 리포트
- 대표용 대시보드
- 부서장용 실행보드
- 감사 로그

## 2. 이번 버전에서 하지 않을 것
- 외부 메신저 연동
- 이메일 자동 발송 고도화
- AI 자동 작성 본격화
- 복합 프로젝트/에픽 구조
- 예산/비용 관리
- 채팅형 협업 기능
- 공개 API 외부 제공

## 3. 화면 잠금 원칙
- 대표 화면은 요약 우선, 상세는 drill-down
- 부서장 화면은 업무 처리 속도 우선
- 담당자 화면은 모바일 증빙 입력 우선
- 모든 상태 변경은 이력으로 남아야 함

## 4. 데이터 잠금 원칙
- 지시사항은 삭제보다 비활성/보존 우선
- 종결 후에도 이력/증빙은 유지
- 상태값은 enum 수준으로 고정
- 관리번호는 중복 불가
- 지연 판정은 규칙 기반 자동 계산

## 5. MVP 기준 통과 조건
- 대표 대시보드에서 총 건수, 신규, 진행, 지연, 종결, 종결률, 기한 준수율 확인 가능
- 지시 상세에서 타임라인, 증빙, 상태 이력 확인 가능
- 부서장은 담당자 배정/상태 업데이트/완료 요청 가능
- TF는 승인/반려 및 결산 생성 가능
