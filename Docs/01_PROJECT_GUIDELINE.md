# 프로젝트 지침서

## 1. 프로젝트 정체성
본 시스템은 대표이사 지시사항을 회사 전체 기준으로 통합 관리하고,
각 부서의 실행과 이행 증빙을 구조화하여 대표가 전사 진행 현황과 세부 이행 근거를
한 번에 확인할 수 있도록 하는 운영 통제 플랫폼이다.

## 2. 해결하려는 문제
- 회의록 본문에 지시사항이 흩어져 전체 현황 파악이 어려움
- 부서별 보고 형식이 달라 비교와 점검이 어려움
- 지시사항 상태가 표준화되지 않아 현재 위치 파악이 어려움
- 주간 기준 신규 / 진행 / 지연 / 종결 결산이 어려움
- 종결 판단 기준이 불명확하여 관리 일관성이 떨어짐

## 3. 운영 철학
- 말이 아니라 증빙으로 관리한다.
- 회의 내용이 아니라 지시 이행을 관리한다.
- 부서별 자율 실행은 보장하되 상태 기준과 보고 기준은 통일한다.
- 대표에게는 요약을, 실무에는 세부 운영 도구를 제공한다.
- 기록은 누적되어 향후 조직 운영 데이터 자산이 된다.

## 4. 변경 불가 핵심 구조
1. 전사 총괄 관리
2. 부서별 분류 관리
3. 상태값 표준화
4. 행동 로그 및 증빙 관리
5. 완료 요청 및 종결 승인
6. 주간 결산 자동 생성

## 5. MVP 범위
- 지시사항 등록
- 관리번호 자동 부여
- 부서 및 책임자 배정
- 상태값 관리
- 행동 로그 등록
- 사진/문서 증빙 첨부
- 지연 자동 감지
- 완료 요청 및 승인/반려
- 대표 대시보드
- 부서장 실행보드
- 주간 결산 리포트

## 6. 제외 범위
- 외부 ERP/그룹웨어 연동
- 고급 AI 예측 모델
- 다국어 지원
- 복잡한 OKR/성과평가 연동
- 전자결재 전체 대체
- 대외 고객용 포털

## 7. 성공 기준
- 대표 지시사항의 70% 이상을 시스템에서 등록/추적
- 지연 감지가 수작업 없이 가능
- 주간 보고 준비 시간이 절반 이하로 감소
- 부서별 업데이트 기준이 통일됨
- 종결 판단이 승인 기반으로 일관화됨

## 8. 문서 우선순위
설계 분쟁 시 우선순위는 아래와 같다.
1. 본 지침서
2. 범위 잠금 문서
3. DB 스키마 잠금 문서
4. 화면 정의 문서
5. API 계약 초안

## 9. 향후 확장 방향
- AI 요약
- 지연 예측
- 회의록 자동 연결
- 대표 브리핑 초안 생성
- 부서별 운영 인사이트 분석
