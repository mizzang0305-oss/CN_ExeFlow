# 제품 정의서(PRD)

## 1. 제품명
DirectiveOps (가칭)
한글 명칭: 대표 지시 이행관리 시스템

## 2. 한 줄 정의
대표 지시를 전사 기준으로 관리하고, 부서 실행과 증빙을 추적하여
결산 중심 의사결정을 가능하게 하는 운영 플랫폼

## 3. 핵심 사용자
- 대표
- TF 총괄 관리자
- 부서장
- 담당자
- 조회 전용 사용자(감사/참조)

## 4. 사용자별 핵심 니즈
### 대표
- 전체 진행 상황을 한눈에 보고 싶다
- 지연 건, 마감 임박 건, 중요 건을 빨리 파악하고 싶다
- 필요 시 근거 자료까지 즉시 내려가 보고 싶다

### TF 총괄
- 지시사항 번호를 부여하고 관리대장을 운영하고 싶다
- 부서별 업데이트를 표준 형식으로 취합하고 싶다
- 주간 결산표를 자동에 가깝게 만들고 싶다

### 부서장
- 우리 부서에 배정된 지시사항만 모아 보고 싶다
- 담당자를 배정하고 진행 현황을 업데이트하고 싶다
- 완료 요청 전에 빠진 증빙을 확인하고 싶다

### 담당자
- 현장에서 빠르게 행동 로그와 사진을 올리고 싶다
- 무엇을 언제까지 해야 하는지 명확히 알고 싶다

## 5. 해결 방식
- 지시사항을 개별 데이터로 관리
- 부서와 담당자 연결
- 행동 로그와 증빙 첨부
- 상태 머신 기반으로 신규/진행/지연/완료요청/종결/반려 표준화
- 주간 결산 자동 집계

## 6. 주요 페이지
- 대표 대시보드
- 지시사항 목록/상세/등록
- 부서 실행보드
- 행동 로그 및 증빙 등록
- 주간 결산 리포트
- 기준정보 관리(부서/사용자/상태 기준)

## 7. 비기능 요구사항
- 모바일 입력 우선
- 파일 첨부 안정성
- 감사 로그 보존
- 권한별 화면 분리
- 검색/필터 빠른 응답
- 상태 변경 이력 보존
