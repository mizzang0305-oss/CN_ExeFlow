# 비즈니스 룰 잠금 문서

## 1. 관리번호 규칙
- 형식: DIR-YYYY-0001
- 연도별 시퀀스 증가
- 중복 불가
- 발급 후 변경 불가

## 2. 상태 변경 룰
- 상태 변경은 권한자만 가능
- 상태 변경 시 audit_logs 기록 필수
- COMPLETED 상태는 승인 없이는 진입 불가

## 3. 완료 요청 룰
- 부서장 또는 총괄 관리자만 완료 요청 가능
- 최소 1건 이상의 실행 로그가 있어야 함
- 핵심 증빙 누락 시 요청 불가

## 4. 종결 룰
- 총괄 승인 필수
- closed_at 기록 필수
- close_reason 입력 권장
- 종결 후 수정은 제한적 허용 + 감사 로그 기록

## 5. 지연 룰
- due_date 경과 후 COMPLETED 아님 = 지연
- 지연은 수동 설정보다 자동 계산 우선
- 지연 상태 해제 시 사유 기록

## 6. 삭제 룰
- 지시사항 물리 삭제 금지
- 첨부 삭제는 소프트 삭제 또는 별도 로그 필요
- 감사 추적을 해치면 안 됨

## 7. 파일 첨부 룰
- 이미지, 문서 허용
- 파일명, 크기, MIME 보존
- 업로더 정보 기록 필수

## 8. 리포트 룰
- 주간 리포트는 동일 주차 중복 생성 금지
- 생성 시점 데이터 스냅샷 보존
- 생성 후 수정 시 버전 이력 고려

## 9. 권한 룰
- CEO는 전사 조회 가능
- 부서장은 자기 부서 수정 가능
- STAFF는 배정된 업무 중심 수정 가능
- VIEWER는 조회만 가능
